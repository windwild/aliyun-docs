## **问题描述**

请求中缺少`AccessKeyId`参数。

## **问题原因**

您发起了在URL中兼容Amazon S3的请求，但是请求中缺少`AWSAccessKeyId`参数。

## **问题示例**

您发起了在URL参数中携带签名的请求，但是请求中缺少`AWSAccessKeyId`参数。

```http
GET /test.txt?Expires=1141889120Signature=vjbyPxybdZaNmGa%2ByT272YEAiv**** HTTP/1.0
Date: Tue, 20 Dec 2022 08:48:18 GMT
Host: oss-example.oss-cn-hangzhou.aliyuncs.com
x-oss-s3-compat: true
```

## **解决方案**

发起Amazon S3兼容的URL请求包含3个必选字段，分别为`AWSAccessKeyId`，`Expires`以及`Signature`。正确示例如下：

```http
GET /test.txt?AWSAccessKeyId=nz2p****&Expires=1141889120&Signature=vjbyPxybdZaNmGa%2ByT272YEAiv**** HTTP/1.0
Date: Tue, 20 Dec 2022 08:48:18 GMT
Host: oss-example.oss-cn-hangzhou.aliyuncs.com
x-oss-s3-compat: true
```

推荐使用Amazon S3 SDK发起请求。更多信息，请参见 [使用Amazon S3 SDK访问OSS](https://help.aliyun.com/zh/oss/developer-reference/use-aws-sdks-to-access-oss "")。
