## **问题描述**

ListObject请求中list-type参数有误。

## **问题原因**

您发起了存储空间列举请求，但是请求中的list-type参数不正确。

## **问题示例**

比如您发起了如下请求：

```http
GET /?list-type=test HTTP/1.1
Host: BucketName.oss-cn-hangzhou.aliyuncs.com
Date: GMT Date
Authorization: SignatureValue
```

上述请求list-type参数为test，但是该接口限制list-type参数只能为2，因此会返回该错误。

## **解决方案**

如果您希望使用ListObject接口列举存储空间，可以直接省略list-type参数：

```http
GET / HTTP/1.1
Host: BucketName.oss-cn-hangzhou.aliyuncs.com
Date: GMT Date
Authorization: SignatureValue
```

如果您希望使用ListObjectV2接口列举存储空间，则需要把list-type参数值设置为2：

```http
GET /?list-type=2 HTTP/1.1
Host: BucketName.oss-cn-hangzhou.aliyuncs.com
Date: GMT Date
Authorization: SignatureValue
```
