### **多个应用** **或Job任务**\
\
复杂场景下，您的SAE可能运行多个应用或Job任务。计算费用的步骤如下：\
\
**重要**\
\
相同版本的应用需要合并计算费用，不同版本的应用需要分开计算费用。\
\
例如，您的SAE同时运行若干轻量版应用和专业版应用，则应先合并计算轻量版应用的费用，再合并计算专业版应用的费用。\
\
1. 计算 **原始资源使用量**（ **CPU使用量**、 **内存使用量** 和 **磁盘使用量**）。在统计资源的使用时长时，不足1秒的部分按照1秒进行统计。\
\
   - `CPU使用量 = ∑实例数 * 单实例CPU规格（核）* 本月运行时长（秒）`\
\
   - `内存使用量 = ∑实例数 * 单实例内存规格（GB）* 本月运行时长（秒）`\
\
   - `磁盘使用量 = ∑实例数 *（实例磁盘规格（GiB）- 免费磁盘规格（20GiB））* 本月运行时长（秒）`\
2. 计算 **CU使用量**。\
\
   - `CU使用量 = ∑各类资源使用量 * 相应的CU转换系数`\
3. 如果计费方式是 [资源包（预付费）](https://help.aliyun.com/zh/sae/product-overview/billing-new#ded30bf3c034j "")，则直接从CU资源包中抵扣CU使用量。如果计费方式是 [按量计费（后付费）](https://help.aliyun.com/zh/sae/product-overview/billing-new#e8eb00f3c0yvc "")，则需要按照当月累计的CU使用量（相同版本的应用合并累计，不同版本的应用分开累计），确定CU计费的单价。\
\
   - `总费用 = ∑各阶梯内CU使用量 * 相应的CU阶梯计费单价`\

#### **为什么停止使用资源后，查看账单却发现SAE仍在消费？**\
\
假设您在13:00~14:00使用了SAE资源，在14:00停止使用，并立刻前往 **明细账单** 页签，却查询到SAE在14:00后仍存在消费记录。\
\
因SAE出账周期有2小时左右的延迟，该情况属于正常现象。因此，您在14:00停止使用SAE资源后，请于16:00左右前往 **明细账单** 页签查看账单详情，并按需在 **查看用量明细** 页签导出详细数据。\

## **价格计算器**\
\
为了帮助您合理地规划预算，SAE为您提供了价格计算器，帮助您更准确地预测和控制SAE开销。\
\
### **为什么需要价格计算器？**\
\
- **精准预算管理：** 通过预估不同配置下的费用情况，让您在项目初期就能做好财务规划。\
\
- **透明化成本结构：** 清晰展示各项服务的具体收费细节，增强成本透明度。\
\
\
### **如何使用价格计算器？**\
\
