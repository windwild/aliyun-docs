如果您的网站因遭受恶意CC攻击导致响应缓慢，可配置频次控制功能，CDN节点能够有效识别异常的高频访问，快速阻断访问该网站的恶意请求，提升网站的安全性。

## 启用频次控制

1. 登录 [CDN控制台](https://cdn.console.aliyun.com/)。

2. 在左侧导航栏，单击 **域名管理**。

3. 在 **域名管理** 页面，找到目标域名，单击 **操作** 列的 **管理**。

4. 在指定域名的左侧导航栏，单击 **安全配置**。

5. 单击 **频次控制** 页签。

6. 打开 **频次控制设置** 开关。

7. 在 **频次控制** 对话框，打开 **参数检测** 开关，并选择 **控制模式**。



![频次控制设置](https://help-static-aliyun-doc.aliyuncs.com/assets/img/zh-CN/0009068951/p71196.png)






| **参数** | **说明** |
| --- | --- |



|     |     |
| --- | --- |
| **参数** | **说明** |
| **参数检测** | 开启参数检测，频次控制规则中的URI会带上完整的参数进行匹配。参数检测仅与URI匹配相关，与自定义规则中的匹配规则无关。<br>**说明**<br>**参数检测** 仅适用于自定义规则。 |
| **控制模式** | 您可以选择以下控制模式： <br>   - **正常**<br>     <br>     默认频次控制模式。当您的网站流量无明显异常时，采用该模式，避免被误杀。<br>     <br>   - **紧急**<br>     <br>     当您的网站响应缓慢，且流量、CPU、内存等指标异常时，采用该模式。<br>     <br>   - **自定义**<br>     <br>     您可以根据业务需求自定义防护规则，有效识别异常的高频访问，边缘抵御CC攻击。配置自定义规则的方法，请参见 [自定义频次控制规则](https://help.aliyun.com/zh/cdn/configure-rate-limiting#section-gx1-mbe-z6z "")。 |

8. 单击 **确定**。


## 自定义频次控制规则

**重要**

- 当 **控制模式** 选择 **自定义** 时，需配置自定义规则，其他 **控制模式** 不需要配置自定义规则。

- 最多支持添加5条自定义规则。


1. 单击 **自定义规则** 对应的 **添加规则**。

2. 根据界面提示和下表配置自定义规则。



![频次控制自定义规则](https://help-static-aliyun-doc.aliyuncs.com/assets/img/zh-CN/7906918061/p75891.png)






| **参数** | **说明** |
| --- | --- |



|     |     |
| --- | --- |
| **参数** | **说明** |
| **规则名称** | - 长度为4~30个字符，支持英文、数字。<br>     <br>   - 同一个域名的规则名称不可重复。 |
| **URI** | 指定需要防护的具体地址，例如`/register`。如果地址中包含了参数，例如`/user?action=login`，需打开参数检测开关才会生效。 |
| **匹配方式** | 匹配方式默认按照完全匹配、前缀匹配、模糊匹配的顺序排序并执行，您可以在同类规则中调整优先级，优先级按列表顺序排序，执行规则时按照优先级进行执行。 <br>   - **完全匹配**<br>     <br>     即精确匹配，请求地址必须与配置的URI完全一样才会被统计。<br>     <br>   - **前缀匹配**<br>     <br>     即包含匹配，只要是请求的URI以此处配置的URI开头就会被统计。例如，如果设置URI为`/register`，则`/register.html`会被统计。<br>     <br>   - **模糊匹配**<br>     <br>     <br>     <br>     即根据表达式匹配，当请求的URI与此处的表达式匹配就会被统计。支持用英文句号（.）和星号（\*）匹配：<br>     <br>     <br>     <br>     - 英文句号（.）：表示匹配任意单个字符。<br>       <br>     - 星号（\*）：表示匹配任意重复字符。 |
| **检测及阻断对象** | 频次控制支持的检测对象如下： <br>   - **源IP**<br>     <br>   - **请求Header中指定字段**<br>     <br>   - **访问域名**<br>     <br>   - **请求URL中指定参数** |
| **检测时长** | 指定统计访问次数的周期，需要和检测对象配合。检测时长为10s~600s（包含10s和600s）。 |
| **匹配规则** | 您可以单击 **添加规则**，配置规则的 **类型**、 **参数**、 **逻辑符** 和 **值**。<br>**说明**<br>频次控制命中匹配规则的请求次数是基于单节点进行统计，实际拦截策略生效会滞后，建议您增加访问频次，以便更快触发拦截规则。 |
| **阻断类型** | 指定触发条件后的操作，可以是 **封禁** 或 **人机识别**。<br>   - **封禁**<br>     <br>     触发条件后直接断开连接，所有请求返回403。<br>     <br>   - **人机识别**<br>     <br>     触发条件后用重定向的方式访问客户端（返回200状态码），且系统会自动识别正常访问和攻击，对于攻击行为进行封禁，只有验证通过后才放行。<br>     <br>     例如，单个IP在20秒内访问超过5次，则进行人机识别判断，在10分钟内该IP的访问请求都需要通过人机识别，如果被识别为非法将会被拦截，只有被识别为合法才会放行。 |
| **阻断时长** | 指定执行阻断动作的时间，阻断时间大于等于60秒。 |

3. 单击 **确定**。


## 自定义规则配置示例

存在以下异常场景时，您可以参考配置示例设置，会及时阻断异常请求。

| **异常场景** | **检测对象** | **检测时长** | **匹配规则** | **阻断类型** | **阻断时长** | **结果** |
| --- | --- | --- | --- | --- | --- | --- |

|     |     |     |     |     |     |     |
| --- | --- | --- | --- | --- | --- | --- |
| **异常场景** | **检测对象** | **检测时长** | **匹配规则** | **阻断类型** | **阻断时长** | **结果** |
| 4xx/5xx异常 | IP | 10秒 | `"status_ratio|404">60%`&&`"count">50` | 封禁 | 10分钟 | 某访问IP的返回404状态码占比大于60%，且访问次数大于50次时，进行IP封禁，在10分钟内断开该IP的连接，所有请求返回403。 |
| QPS异常 | 域名 | 10秒 | `"count">N`<br>**说明**<br>_N_表示可以取任意值，您可以根据实际业务需求设置。 | 人机识别 | 10分钟 | 某域名访问次数大于_N_次时，进行人机识别判断，在10分钟内该域名的访问请求都需要通过人机识别，如果被识别为非法将会被拦截，只有被识别为合法才会放行。 |
